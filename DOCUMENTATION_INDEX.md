## üîÑ Documentation Workflow

### When Adding Features

1. **Update API_REFERENCE.md** if adding public API
2. **Update DEVELOPMENT_GUIDE.md** if adding patterns
3. **Update README.md** if major feature
4. **Update CHANGELOG.md** always
5. **Update QUICK_REFERENCE.md** if commonly used

### When Fixing Bugs

1. **Update CHANGELOG.md** with fix
2. **Update documentation** if bug revealed incorrect docs
3. **Add to troubleshooting** if common issue

### When Refactoring

1. **Update API_REFERENCE.md** if APIs changed
2. **Update architecture docs** if structure changed
3. **Update CHANGELOG.md** with changes
